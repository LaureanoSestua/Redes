Formulario sobre HTTP

  • 1) ¿Sobre qué protocolo binario está montado el protocolo HTTP?

    HTTP está montado sobre el protocolo binario TCP/IP.

  • 2) ¿Cuáles son los clientes HTTP y los servidores HTTP más utilizados?

    Los clientes HTTP más utilizados son curl, Postman y navegadores como Chrome, mientras que los servidores HTTP más comunes incluyen Apache, Nginx y IIS.

  • 3) ¿Qué verbos admite un comando en el requerimiento HTTP?

    Los verbos HTTP son GET, POST, PUT, DELETE, PATCH, OPTIONS, y HEAD.

  • 4) ¿Qué contenido lleva el body de un requerimiento HTTP?

    El body de un requerimiento HTTP puede contener datos en formatos como JSON, XML, o datos de formularios.

  • 5) ¿Qué diferencia existe entre un URL y un URI?

    Un URL es un tipo de URI que especifica cómo localizar un recurso. Un URI es un identificador más general.

  • 6) ¿Cómo se almacena información relacionada con las respuestas HTTP?

    La información relacionada con las respuestas HTTP se almacena en encabezados (headers), cookies y en el cache del navegador.

  • 7) ¿Qué significa Virtual hosting?

    El Virtual hosting permite que un servidor maneje múltiples dominios (o sitios web) desde una sola dirección IP.

  • 8) ¿Por qué el HTTP se considera stateless?

    HTTP es stateless porque cada solicitud es independiente y no guarda estado entre diferentes peticiones.

  • 9) ¿Qué nueva versión de HTTP se está usando para mejorar la velocidad de la web?

    La nueva versión de HTTP es HTTP/2, diseñada para mejorar la velocidad de la web mediante multiplexación y compresión de encabezados.

  • 10) ¿Cómo va a mejorar el comportamiento de una aplicación web en ambientes ruidosos o de débil conexión?

    En ambientes ruidosos o de débil conexión, HTTP/2 y HTTP/3 mejoran el comportamiento mediante la reducción de latencia y el manejo más eficiente de las conexiones.
